# PARA — Organizing Information for Action

**Author:** Pi (2nd Brain system documentation)  
**Date:** 2026-05-09  
**Source:** [raw/gists/PARA-method.md](../../raw/gists/PARA-method.md)

## Summary

PARA is a universal organizational framework developed by Tiago Forte that sorts information by **actionability** rather than by subject matter. The four categories — Projects, Areas, Resources, Archives — map directly onto the lifecycle of raw sources in our 2nd Brain system. PARA governs what is *active now*, while the Zettelkasten governs how knowledge is *connected over time*. This note documents both the original framework and our system's adaptation of it.

## Key Claims

1. **Classify by actionability, not by subject.** A "project" has a deadline and outcome; an "area" is a responsibility without end. Most people sort by topic (books, links, notes) which buries urgency.
2. **Four universal categories across all tools.** The same four folders (P, A, R, A) apply to your notes app, file system, email, and task manager.
3. **Our system applies PARA to the raw-source lifecycle, not the wiki itself.** Raw sources move through active processing (projects) → permanent categories (areas) → ingested into wiki (resources) → stale/deprecated (archives).
4. **The wiki is organized by Zettelkasten principles, not PARA.** This is a deliberate separation: PARA for workflow, Zettelkasten for structure.

## Synthesis Notes

This note serves as the canonical system reference for why raw/ is structured as immutable category directories ( Areas) and how sources transition through the PARA lifecycle. It also clarifies a common confusion: the wiki itself does not use PARA folders. PARA is for *workflow*; the Zettelkasten is for *knowledge architecture*. This pairing is intentional and documented here to prevent future LLM sessions from imposing a folder hierarchy on the wiki.
